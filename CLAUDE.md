# CLAUDE.md

This file gives Claude Code the project-specific instructions for evolving this repository.

## Project goal

Build a personal interactive notebook inspired by the visual language of Death Note: a black ominous notebook, page-turning interaction, aged rule pages, handwritten/written entries, and a dark supernatural mood.

This is a personal study project, but the repository is public. Do not add copyrighted scans, screenshots, official manga pages, anime frames, official logos as image assets, or copied rule text from the manga/anime.

Use references only as visual inspiration and recreate the look with CSS, layout, typography, and original text.

## Reference context

Death Note is a manga written by Tsugumi Ohba and illustrated by Takeshi Obata, originally serialized by Shueisha in Weekly Shonen Jump. The anime adaptation was produced by Madhouse and aired in 2006-2007. Because the work is a published commercial manga/anime franchise, treat official artwork, screenshots, pages, logo images, and original rule text as protected material.

When improving the UI, aim for an inspired implementation rather than a direct asset copy.

## Visual direction

The notebook should feel much closer to the original prop:

- Black matte cover, almost leather-like, with subtle worn texture.
- Large white serif/gothic title centered on the cover.
- Strong contrast: black cover, white title, minimal decoration.
- Aged yellow/brown pages with stains, darkened edges, subtle paper grain, and old-book shadows.
- Rules page before editable pages.
- Serif typography for rules and page writing.
- Sparse, ominous layout. Avoid colorful UI.
- Avoid modern card/dashboard aesthetics.
- The experience should feel like opening an artifact, not a note-taking SaaS app.

## Copyright-safe implementation rules

Do:

- Recreate texture with CSS gradients, shadows, pseudo-elements, SVG noise generated by us, or custom procedural assets.
- Write original fictional rules inspired by the concept and UI structure.
- Use open/free fonts or system fonts that approximate the mood.
- Use original names for internal domain concepts when possible, such as `Notebook`, `NotePage`, `RulesPage`, and `PageCard`.
- Keep reference images out of the repository if they are official or fan-owned.

Do not:

- Commit manga scans, screenshots from anime episodes, official posters, official logos, or copied panels.
- Copy the official Death Note rules verbatim.
- Add downloaded images from Google/Pinterest/Fandom/marketplaces as project assets.
- Add a `public/references` folder with copyrighted material.
- Present this as an official Death Note product.

If local visual references are needed, keep them ignored by git.

Recommended `.gitignore` entry if references are used locally:

```gitignore
public/references/
references/
```

## Architecture rules

This project is a single Next.js app, not a monorepo.

Keep the current Clean Architecture / DDD boundaries:

```txt
src/
  app/                 # Next.js App Router
  core/                # Domain entities, value objects, errors, ports
  application/         # Use cases, DTOs, mappers
  infra/               # Concrete adapters, storage implementations
  features/            # React hooks/components for feature UI
```

Dependency direction:

```txt
features -> application -> core
infra -> application/core contracts
```

Domain code must not depend on React, Next.js, browser APIs, CSS, or UI libraries.

## UI implementation priorities

### 1. Cover

Improve `CoverPage` so it resembles a black supernatural notebook cover:

- Full black cover surface.
- Centered title in large white serif/gothic text.
- Subtle leather/paper texture using CSS only.
- Worn edges and inner shadows.
- Optional faint scratches or diagonal grain using pseudo-elements.
- No official image logo.

### 2. Rules page

Improve `RulesPage`:

- It should be the first internal page.
- Use original rule text.
- Make it look like old printed rules, not a modern list.
- Use numbered rules, serif text, long line height, and aged paper background.
- Do not copy the original rules verbatim.

### 3. Editable pages

Improve `PageCard`:

- Use aged paper visual treatment.
- Make the textarea feel embedded in the paper.
- Use serif typography and subtle line guides.
- Keep accessibility labels.
- Keep character limit and character counter.

### 4. Page turning

Keep `react-pageflip` for desktop.

Requirements:

- Cover page first.
- Rules page second.
- User-created note pages after rules.
- Mobile fallback should remain usable.
- Do not break localStorage persistence.

### 5. Atmosphere

The page around the notebook should be dark, minimal, and cinematic:

- Dark radial gradients.
- Low red/brown accents.
- No bright UI colors.
- Buttons should feel like controls around an artifact, not SaaS buttons.

## Code quality guidelines

- Prefer small components.
- Keep UI-specific logic in `features/death-note`.
- Keep notebook business rules in `core` and `application`.
- Do not put localStorage access in domain or application use cases. Keep it in `infra`.
- Avoid overengineering; this is still a personal project.
- Keep TypeScript strict.
- Avoid `any` unless absolutely necessary and documented.
